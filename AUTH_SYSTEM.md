# نظام المصادقة الجديد - منصة نحكي

## نظرة عامة
تم تحديث نظام المصادقة في منصة نحكي ليصبح أكثر مرونة وسهولة في الاستخدام، مع الحفاظ على الأمان.

## التغييرات الجديدة

### 🔐 تسجيل الدخول
- **بكلمة المرور**: المستخدم يمكنه تسجيل الدخول باستخدام اسم المستخدم/البريد الإلكتروني/رقم الهاتف + كلمة المرور
- **بـ OTP (اختياري)**: إذا ترك المستخدم حقل كلمة المرور فارغاً، سيتم إرسال OTP للتحقق

### 📝 إنشاء حساب جديد
- يتطلب جميع البيانات الشخصية + كلمة المرور
- يتم إرسال OTP للتحقق من الحساب الجديد
- لا يتم تفعيل الحساب إلا بعد التحقق من OTP

### 🔑 نسيان كلمة المرور
- المستخدم يدخل اسم المستخدم/البريد الإلكتروني/رقم الهاتف
- يتم إرسال OTP لإعادة تعيين كلمة المرور
- بعد التحقق من OTP، يمكن تعيين كلمة مرور جديدة

## المزايا الجديدة

### ✅ تحسين تجربة المستخدم
- تسجيل دخول سريع بكلمة المرور للاستخدام اليومي
- خيار OTP متاح للأمان الإضافي
- نظام "نسيت كلمة المرور؟" سهل الاستخدام

### 🛡️ الأمان
- تشفير كلمات المرور باستخدام bcrypt
- OTP محدود الوقت (10 دقائق لإعادة التعيين، 5 دقائق للتسجيل)
- التحقق من قوة كلمة المرور
- خيار "البقاء على تسجيل الدخول" (365 يوماً)

### 🔧 المرونة
- دعم تسجيل الدخول بـ اسم المستخدم أو البريد الإلكتروني أو رقم الهاتف
- خيارات متعددة للمصادقة

## API Endpoints

### تسجيل الدخول
```
POST /api/auth/login
Body: { identifier: string, password: string }
```

### إنشاء حساب
```
POST /api/auth/register
Body: { username, email, phone, password, firstName, lastName, ... }
```

### نسيان كلمة المرور
```
POST /api/auth/forgot-password
Body: { identifier: string }
```

### إعادة تعيين كلمة المرور
```
POST /api/auth/reset-password
Body: { identifier: string, otp: string, newPassword: string }
```

### إرسال OTP
```
POST /api/auth/send-otp
Body: { identifier: string }
```

### التحقق من OTP
```
POST /api/auth/verify-otp
Body: { identifier: string, otp: string }
```

## متطلبات كلمة المرور
- 6 أحرف على الأقل
- يُفضل استخدام مزيج من الأحرف الكبيرة والصغيرة والأرقام والرموز
- عرض مؤشر قوة كلمة المرور في الوقت الفعلي

## الحالات المختلفة

### 1. المستخدم الجديد
1. ملء نموذج التسجيل بجميع البيانات
2. استلام OTP للتحقق
3. إدخال OTP لتفعيل الحساب
4. تسجيل الدخول التلقائي

### 2. المستخدم الحالي - تسجيل دخول عادي
1. إدخال اسم المستخدم/البريد/الهاتف + كلمة المرور
2. تسجيل الدخول الفوري

### 3. المستخدم الحالي - تسجيل دخول بـ OTP
1. إدخال اسم المستخدم/البريد/الهاتف فقط (ترك كلمة المرور فارغة)
2. استلام OTP
3. إدخال OTP للدخول

### 4. نسيان كلمة المرور
1. النقر على "نسيت كلمة المرور؟"
2. إدخال اسم المستخدم/البريد/الهاتف
3. استلام OTP
4. إدخال OTP + كلمة المرور الجديدة
5. تأكيد كلمة المرور
6. العودة لتسجيل الدخول

## ملاحظات تقنية
- يتم حفظ OTP مؤقتاً في الذاكرة (في الإنتاج يُفضل Redis)
- JWT للجلسات مع انتهاء صلاحية 7 أيام
- دعم كامل للعربية مع RTL
- تصميم متجاوب لجميع الأجهزة
