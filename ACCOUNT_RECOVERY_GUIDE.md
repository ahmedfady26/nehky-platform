# دليل ميزة استرداد الحساب في منصة نحكي

## 📋 نظرة عامة
تم تطوير ميزة شاملة لاسترداد الحساب في منصة نحكي، تتيح للمستخدمين استعادة حساباتهم في حالة نسيان كلمة المرور أو اسم المستخدم. تعتمد الميزة على نظام مزدوج من التحقق: رقم الهاتف وإيميل الاسترداد الاختياري.

## 🔧 المكونات المطورة

### 1. قاعدة البيانات
- **إضافة حقل جديد**: `recoveryEmail` (String?, اختياري) في جدول `User`
- **المزامنة**: تم تحديث قاعدة البيانات عبر `prisma db push`

### 2. واجهات TypeScript
```typescript
// تحديث User interface
interface User {
  // ...existing fields
  recoveryEmail?: string; // إيميل الاسترداد الاختياري
}

// تحديث RegisterForm interface
interface RegisterForm {
  // ...existing fields
  recoveryEmail?: string; // إيميل الاسترداد الاختياري
}
```

### 3. صفحة التسجيل (`/src/app/auth/page.tsx`)
- **حقل جديد**: "إيميل الاسترداد (اختياري)" في نموذج التسجيل
- **نموذج استرداد الحساب**: يظهر عند النقر على "استعادة الحساب"
- **منطق الاسترداد**: إدخال رقم الهاتف → إرسال OTP → إشعار بإرسال إيميل الاسترداد

### 4. API التسجيل (`/src/app/api/auth/register/route.ts`)
- **استقبال recoveryEmail**: التحقق من صحة الإيميل إذا تم إدخاله
- **حفظ في قاعدة البيانات**: تخزين إيميل الاسترداد مع بيانات المستخدم

### 5. API استرداد الحساب (`/src/app/api/auth/account-recovery/route.ts`)
```typescript
// المسار الجديد
POST /api/auth/account-recovery

// البيانات المطلوبة
{
  "phone": "رقم الهاتف"
}

// الاستجابة
{
  "success": true,
  "message": "تم إرسال رمز التحقق إلى رقم هاتفك",
  "hasRecoveryEmail": true,
  "recoveryEmailSent": true,
  "maskedRecoveryEmail": "te***@gmail.com"
}
```

**العمليات التي يقوم بها:**
1. البحث عن المستخدم برقم الهاتف
2. إنشاء OTP وحفظه مؤقتاً
3. إرسال OTP عبر SMS
4. إرسال إيميل استرداد (إذا كان متوفراً) يحتوي على معلومات الحساب

### 6. خدمة الإيميل (`/src/lib/email-service.ts`)
- **إرسال إيميل الاسترداد**: `sendAccountRecoveryEmail()`
- **إرسال OTP عبر SMS**: `sendOTPViaSMS()`
- **تصميم إيميل جميل**: HTML responsive مع شعار المنصة
- **جاهز للإنتاج**: يمكن ربطه بـ SendGrid أو AWS SES

### 7. صفحة تعديل الملف الشخصي (`/src/app/profile/edit/page.tsx`)
- **إعدادات الأمان**: قسم مخصص لإيميل الاسترداد
- **تعديل الإيميل**: يمكن للمستخدم إضافة/تعديل/حذف إيميل الاسترداد
- **تحديث آمن**: التحقق من صحة البيانات قبل الحفظ

### 8. API الملف الشخصي (`/src/app/api/user/profile/route.ts`)
- **GET**: استرجاع بيانات المستخدم (متضمنة إيميل الاسترداد)
- **PUT**: تحديث بيانات المستخدم (متضمنة إيميل الاسترداد)
- **التحقق من الأمان**: استخدام نظام التحقق المحسن

### 9. نظام التحقق المحسن (`/src/lib/auth-middleware.ts`)
- **التحقق من JWT**: `verifyAuthToken()`
- **استخراج بيانات المستخدم**: `getUserFromRequest()`
- **التحقق من الأدوار**: `hasRequiredRole()`, `isInfluencer()`, `isTopFollower()`
- **حماية المنشورات**: `canAccessPost()`

## 🎯 مسار الاستخدام

### للمستخدم الجديد:
1. **التسجيل**: يملأ النموذج ويضيف إيميل الاسترداد (اختياري)
2. **الحفظ**: يتم حفظ إيميل الاسترداد في قاعدة البيانات

### للمستخدم الحالي:
1. **تعديل الملف الشخصي**: يذهب إلى `/profile/edit`
2. **إعدادات الأمان**: يضيف/يعدل إيميل الاسترداد
3. **الحفظ**: يتم تحديث البيانات

### عند فقدان الحساب:
1. **استرداد الحساب**: ينقر على "استعادة الحساب" في صفحة تسجيل الدخول
2. **إدخال رقم الهاتف**: يدخل رقم الهاتف المرتبط بالحساب
3. **استقبال OTP**: يصله رمز التحقق عبر SMS
4. **إيميل الاسترداد**: يصله إيميل يحتوي على معلومات الحساب (اسم المستخدم، رقم الهاتف)

## 🔒 الأمان والحماية

### حماية البيانات:
- **تشفير كلمات المرور**: تستخدم bcrypt
- **JWT آمن**: رموز تنتهي صلاحيتها خلال أسبوع
- **OTP مؤقت**: ينتهي خلال 10 دقائق
- **إخفاء الإيميل**: يتم إخفاء جزء من إيميل الاسترداد (`te***@gmail.com`)

### التحقق من الصحة:
- **تفرد البيانات**: فحص تفرد اسم المستخدم، الإيميل، رقم الهاتف
- **صحة الإيميل**: التحقق من صيغة الإيميل باستخدام regex
- **البيانات المطلوبة**: التأكد من وجود البيانات الأساسية

## 🚀 الإعداد للإنتاج

### متغيرات البيئة المطلوبة:
```env
# JWT Secret
JWT_SECRET=your-super-secret-jwt-key

# إعدادات SMTP (لإرسال الإيميلات)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
FROM_EMAIL=noreply@nehky.com

# إعدادات SMS (لإرسال OTP)
SMS_PROVIDER_API_KEY=your-sms-api-key
SMS_PROVIDER_URL=https://api.sms-provider.com/send
```

### خدمات الإنتاج الموصى بها:
- **إرسال الإيميلات**: SendGrid, AWS SES, Mailgun
- **إرسال SMS**: Twilio, AWS SNS, خدمة محلية
- **قاعدة البيانات**: PostgreSQL, MySQL
- **التخزين المؤقت**: Redis (للـ OTP)
