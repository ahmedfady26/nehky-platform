# نظام استعادة الحساب الآمن - الرابط المؤقت

## نظرة عامة

تم تطوير نظام استعادة حساب آمن ومتقدم لمنصة "نحكي" يعتمد على **رابط مؤقت لإعادة تعيين كلمة المرور** بدلاً من إرسال كلمة سر مؤقتة أو OTP، مما يوفر مستوى أمان أعلى وتجربة مستخدم أفضل.

## ✨ الميزات الجديدة

### 🔐 الأمان المتقدم
- **رابط مؤقت آمن**: بدلاً من OTP قصير، يتم توليد رمز تحقق طويل (64 حرف hex)
- **صالحية محدودة**: الرابط صالح لمدة 15 دقيقة فقط
- **استخدام واحد**: يتم إبطال الرابط فور الاستخدام أو انتهاء الوقت
- **عدم كشف المعلومات**: لا يتم الكشف عن وجود/عدم وجود الحساب
- **Rate Limiting**: حماية من المحاولات المتكررة

### 📧 نظام الإيميل المتقدم
- **تصميم احترافي**: إيميل HTML متجاوب مع تدرج الألوان
- **دعم كامل للعربية**: RTL + خطوط مناسبة
- **تعليمات واضحة**: إرشادات أمان مفصلة
- **تحذيرات أمنية**: تنبيهات عدم مشاركة الرابط

### 🎯 أولوية الإرسال الذكية
1. **إيميل الاسترداد** (إذا كان متاحاً)
2. **البريد الداخلي** `username@nehky.com`
3. **البريد الخارجي** (كخيار أخير)

## 🏗️ الهيكل التقني

### APIs المطورة

#### 1. `/api/auth/recover-account` (محدث)
**الوظيفة**: بدء عملية الاسترداد وإرسال الرابط
```json
{
  "method": "email" | "phone",
  "identifier": "string"
}
```

**التحديثات**:
- توليد رمز آمن بـ `crypto.randomBytes(32).toString('hex')`
- حفظ البيانات في `resetTokens` بدلاً من `tempOTPs`
- إرسال إيميل HTML متقدم

#### 2. `/api/auth/verify-reset-token` (جديد)
**الوظيفة**: التحقق من صحة الرابط قبل عرض نموذج إعادة التعيين
```
GET /api/auth/verify-reset-token?token=XXXXX
```

**الاستجابة**:
```json
{
  "success": true,
  "message": "رمز الاسترداد صحيح",
  "user": {
    "username": "testuser",
    "fullName": "اختبار المستخدم"
  }
}
```

#### 3. `/api/auth/confirm-recovery` (محدث)
**الوظيفة**: تأكيد الاسترداد وتغيير كلمة المرور
```json
{
  "token": "string",
  "newPassword": "string"
}
```

**التحديثات**:
- استقبال `token` بدلاً من `identifier + otp`
- تعيين الرمز كـ `used: true` بدلاً من حذفه

### الواجهات الجديدة

#### 1. `/reset-password` (جديد)
**صفحة إعادة تعيين كلمة المرور**:
- التحقق التلقائي من صحة الرابط
- عرض معلومات المستخدم للتأكيد
- نموذج تغيير كلمة المرور مع التحقق
- رسائل خطأ وتأكيد واضحة
- تحويل تلقائي لصفحة تسجيل الدخول بعد النجاح

#### 2. `/account-recovery` (محدث)
**التحديثات**:
- إزالة خطوة OTP
- صفحة تأكيد الإرسال مع تعليمات
- رسائل محدثة تشير للرابط بدلاً من الرمز

## 🚀 رحلة المستخدم

### 1. طلب الاسترداد
- المستخدم يذهب لـ `/account-recovery`
- يختار الطريقة (إيميل/هاتف)
- يدخل المعرف المناسب
- يضغط "إرسال رابط الاسترداد"

### 2. استقبال الرابط
- يتلقى إيميل HTML متقدم
- ينقر على رابط إعادة التعيين
- يتم توجيهه لـ `/reset-password?token=XXXXX`

### 3. إعادة التعيين
- التحقق التلقائي من صحة الرابط
- عرض معلومات المستخدم للتأكيد
- إدخال كلمة المرور الجديدة مع التأكيد
- تغيير كلمة المرور وإبطال الرابط
- تحويل لصفحة تسجيل الدخول

## 🛡️ الأمان

### حماية الرمز
- **طول آمن**: 64 حرف hex (256 بت عشوائية)
- **تشفير قوي**: `crypto.randomBytes` الآمن
- **تخزين مؤقت**: في الذاكرة مع انتهاء صلاحية
- **استخدام واحد**: يتم إبطاله فور الاستخدام

### حماية التطبيق
- **Rate Limiting**: 3 محاولات كل 15 دقيقة
- **عدم كشف المعلومات**: رسائل عامة للجميع
- **تحقق شامل**: من صحة البيانات والرمز
- **إبطال تلقائي**: عند انتهاء الوقت

### حماية الاتصال
- **HTTPS إجباري**: في الإنتاج
- **رؤوس أمان**: مع Next.js
- **التحقق المزدوج**: رمز + معرف المستخدم
