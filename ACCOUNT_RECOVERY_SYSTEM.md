# نظام استعادة الحساب - منصة نحكي

## نظرة عامة

تم تطوير نظام استعادة حساب شامل ومتقدم لمنصة نحكي يدعم طريقتين أساسيتين للاسترداد مع حماية كاملة من الهجمات والتلاعب.

## الميزات الرئيسية

### 🔐 طرق الاسترداد المتعددة
1. **الاسترداد عبر البريد الإلكتروني**
   - البريد الداخلي التلقائي: `username@nehky.com`
   - إيميل الاسترداد الاختياري
   - البريد الخارجي (إذا كان متاحاً)

2. **الاسترداد عبر رقم الهاتف**
   - إرسال OTP عبر SMS
   - دعم الأرقام الدولية مع مفاتيح البلدان

### 🛡️ ميزات الأمان

- **عدم كشف المعلومات الحساسة**: لا يتم الكشف عن وجود أو عدم وجود الحساب
- **Rate Limiting**: حماية من المحاولات المتكررة (3 محاولات كل 15 دقيقة)
- **انتهاء صلاحية OTP**: الرموز صالحة لمدة 10 دقائق فقط
- **التحقق من قوة كلمة المرور**: إنجليزية فقط، 6 أحرف كحد أدنى
- **تشفير كلمات المرور**: استخدام bcrypt

## الهيكل التقني

### البيانات المطلوبة في قاعدة البيانات

```typescript
// نموذج User محدث
model User {
  id            String   @id @default(cuid())
  username      String   @unique
  email         String?  @unique
  phone         String   @unique
  internalEmail String   @unique  // البريد الداخلي التلقائي
  recoveryEmail String?  // إيميل الاسترداد الاختياري
  password      String
  // ... باقي الحقول
}
```

### APIs المطورة

#### 1. `/api/auth/recover-account`
**الغرض**: بدء عملية استعادة الحساب
```json
{
  "method": "email" | "phone",
  "identifier": "string"
}
```

#### 2. `/api/auth/confirm-recovery`
**الغرض**: تأكيد الاسترداد وتغيير كلمة المرور
```json
{
  "identifier": "string",
  "otp": "string",
  "newPassword": "string"
}
```

#### 3. `/api/auth/login` (محدث)
**الغرض**: دعم تسجيل الدخول بالبريد الداخلي
- يدعم: `username`, `email`, `phone`, `internalEmail`

#### 4. `/api/auth/register` (محدث)
**الغرض**: إنشاء البريد الداخلي تلقائياً
- ينشئ `internalEmail` بصيغة `username@nehky.com`

## الواجهات

### 1. صفحة استعادة الحساب (`/account-recovery`)
- واجهة متقدمة وسهلة الاستخدام
- دعم كامل للغة العربية (RTL)
- تصميم متجاوب مع تدرج الألوان المميز
- خطوات واضحة: اختيار الطريقة → إدخال OTP → تعيين كلمة مرور جديدة

### 2. صفحة اختبار النظام (`/test-account-recovery`)
- اختبارات شاملة لجميع السيناريوهات
- مراقبة النتائج في الوقت الفعلي
- اختبار Rate Limiting والحماية

## طريقة الاستخدام

### للمستخدمين:

1. **الذهاب لصفحة تسجيل الدخول**
2. **النقر على "نسيت كلمة المرور؟"**
3. **اختيار طريقة الاسترداد**:
   - البريد الإلكتروني (البريد الداخلي أو إيميل الاسترداد)
   - رقم الهاتف
4. **إدخال المعرف المناسب**
5. **إدخال رمز التحقق المُستلم**
6. **تعيين كلمة مرور جديدة**
